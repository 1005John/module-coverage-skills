## 关键经验

### 1. conn_id=0 不可靠
- conn_id=0 经常返回 URC code=1（重连中），无法成功连接
- 必须使用 conn_id=1/2/4/5

### 2. 配置后立刻连接
- MQTTCFG 配置后如果等太久，连接会超时
- 正确流程：配置 → 等 1 秒 → 连接 → 等 URC

### 3. MQTTCFG 是三参数格式
```
AT+MQTTCFG="keepalive",0,60    ✅ 正确
AT+MQTTCFG="keepalive",60      ❌ 错误（conn_id=60 超范围）
```

### 4. MQTTCONN 是异步的
- OK 只表示命令被接受
- 实际结果看 `+MQTTURC: "conn",<id>,<code>`
- code: 0=成功, 1=重连中, 2=断开, 3=拒绝, 6=超时

### 5. 即使 ERROR 也触发覆盖率桩
- 命令返回 ERROR 时，代码路径仍被执行
- 覆盖率桩照样触发
- 所以错误路径的用例也有价值

### 6. 手册中有未测的 MQTTCFG key
- retrans（重传参数）
- willoption（遗嘱选项）
- willpayload（遗嘱内容）
- pingreq/pingresp（心跳）
- encoding（编码格式）
- cached（缓存模式）
- query（查询全部配置）

### 7. 平台认证路径
- platsel=1: OneNet 平台
- platsel=2: Aliyun 平台
- platsel=3: 华为云平台
- 设置 devinfo 后连接会触发认证函数

### 8. DNS 失败触发大量错误处理
- 连接无效 broker（如 invalid.broker.xyz）会触发 DNS 解析失败
- 这个路径会覆盖大量错误处理代码

### 9. 编译禁止使用 DC ALL
- `ML307R.bat DC ALL` 会从 `ps.7z` 归档恢复源码
- 所有插桩文件会被覆盖
- 必须使用 `ML307R.bat DC` 增量编译

### 10. 跨文件变量不能是 static
- 被 extern 引用的覆盖率变量不能是 static
- 否则链接器报 L6218E 错误
